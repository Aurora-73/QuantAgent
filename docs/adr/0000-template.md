# ADR 模板

> 架构决策记录（Architecture Decision Record）模板。

---

## ADR-XXXX: 决策标题

### 状态

**[建议的状态]**：Accepted / Rejected / Deprecated / Superseded

### 上下文

描述做出此决策的背景和约束条件。

### 可选方案

列出所有考虑过的方案，包括优缺点。

| 方案 | 优点 | 缺点 |
|------|------|------|
| 方案A | ... | ... |
| 方案B | ... | ... |
| 方案C | ... | ... |

### 最终选择

明确说明最终选择的方案。

### 理由

详细说明选择该方案的理由，包括：
- 技术优势
- 业务需求匹配度
- 成本效益
- 风险评估

### 影响范围

列出此决策影响的模块、接口和流程。

### 实施计划

如果需要分阶段实施，列出实施步骤。

### 验证方法

如何验证此决策的正确性？

### 参考

相关的文档、链接或讨论。

---

## 状态说明

| 状态 | 说明 |
|------|------|
| **Proposed** | 决策草案，等待评审 |
| **Accepted** | 决策已通过评审，开始实施 |
| **Rejected** | 决策被拒绝 |
| **Deprecated** | 决策已过时，不再推荐使用 |
| **Superseded** | 决策已被新决策取代 |
| **Implemented** | 决策已完成实施 |
