# MCP 工具能力与限制说明

> 生成日期：2026-07-06 | 最后更新：2026-07-07 | 适用场景：帮助 Agent 自主了解 MCP 工具的能力边界，避免运行时才发现限制

---

## 一、工具总览

> 工具数量不手写，事实来源：`python -m mcp_server.server --list-tools`

| 模块 | 功能 |
|------|------|
| **Data Tools** | 行情、历史、因子、指数、板块、日历、市场概况 |
| **Risk Tools** | 回测、压力测试、Brinson、衰减检测、健康检查 |
| **Knowledge Tools** | 日报、事件、Wiki、决策记忆、预测准确率 |

---

## 二、数据覆盖范围

### 2.1 数据粒度

| 数据类型 | 可用 | 说明 |
|----------|------|------|
| **日线 (OHLCV)** | ✅ | 覆盖 CSI300/CSI500 成分股（数量以 db_stats 为准） |
| **分钟线** | ❌ | 暂不支持，无分钟线数据 |
| **Tick 数据** | ❌ | 暂不支持 |
| **基本面** | ⚠️ | `research.financials` 表，数据可能不全 |

### 2.2 指数覆盖

| 指数 | 可用 | 说明 |
|------|------|------|
| 沪深300 (000300) | ✅ | `get_index_data("000300")` |
| 中证500 (000905) | ✅ | `get_index_data("000905")` |
| 科创50 (000688) | ❌ | `get_market_overview` 中已预留但无数据 |
| 行业指数 | ⚠️ | 无原生指数，可通过 `get_sector_index()` 构建等权指数 |
| 概念板块指数 | ⚠️ | 同上，需调用 `get_sector_index()` 实时构建 |

### 2.3 股票覆盖

| 项目 | 说明 |
|------|------|
| 当前跟踪 | CSI300 + CSI500 成分股（数量以 db_stats 为准） |
| 行业分类 | 申万行业 (baostock) / 东方财富概念板块 (AKShare) |
| 搜索方式 | 支持代码搜索 + 中文名称模糊搜索 (`search_tickers`) |

## 三、关键字段质量

| 字段 | 状态 | 说明 |
|------|------|------|
| `close`, `open`, `high`, `low`, `volume` | ✅ | 日线核心字段，数据完整 |
| `pct_change` | ✅ | 已修复 (2026-07-06)，最近数据正常 |
| `turnover` | ⚠️ | baostock 路径有，AKShare 可能为 NULL |
| `amount` | ✅ | 成交额，数据完整 |
| 估值数据 (PE/PB/EPS) | ⚠️ | `get_stock_valuation` 可获取，但非全覆盖 |

## 四、MCP 工具使用注意事项

### 4.1 自动补充数据

数据不存在于 DB 时，工具会自动从 AKShare/baostock 拉取。这可能导致：
- 首次查询某只股票时响应较慢 (3-10 秒)
- AKShare 偶发性连接中断（`RemoteDisconnected`）

### 4.2 写操作工具

| 工具 | 耗时 | 作用 |
|------|------|------|
| `update_data` | 15-30 min | 从数据源拉取最新行情写入 DB |
| `run_daily_research` | 5-15 min | 完整每日研究流程 |
| `run_backtest` | 1-10 min | 回测并入库结果 |

### 4.3 板块工具（新增）

| 工具 | 耗时 | 说明 |
|------|------|------|
| `get_sector_list` | 2-5s | 获取申万行业/概念板块列表 |
| `get_sector_stocks` | 1-3s | 获取板块成分股 |
| `get_sector_index` | 30s-2min | 从成分股日线构建等权指数 |

### 4.4 已知限制

1. **无分钟线数据** — 需要"开盘涨幅"等日内指标时，只能用 `(close-open)/open` 近似
2. **无板块原生指数** — 只能用 `get_sector_index()` 实时构建等权指数
3. **AKShare 间歇性不可用** — 东方财富接口有时连接失败
4. **baostock 未安装** — 本服务器上 baostock 不可用，所有数据走 AKShare

## 五、给 Agent 的使用建议

### 批量查询策略

- **逐只查询** (`get_history`) → 适合 1-5 只股票
- **板块查询** (`get_sector_stocks` + `get_sector_index`) → 适合行业分析
- **直接查 DB** → 适合 20+ 只股票的大规模批量分析（python duckdb 直连）

### 加速技巧

- 已入库的数据秒级返回（从 DuckDB 读取）
- 新股票首次查询自动拉取，后续查询从缓存返回
- 板块指数第二次调用比首次快（成分股数据已入库）
